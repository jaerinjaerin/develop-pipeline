# 00_requirements.md

## 요청 내용 요약

로그인 페이지 구현 요청. 현재 대시보드는 인증 없이 누구나 접근 가능한 상태이며, 사용자 인증 시스템이 전혀 존재하지 않는다. 로그인 페이지를 신규 구현하여 대시보드 접근을 인증된 사용자로 제한한다.

## 작업 유형

**신규 기능** — 인증 시스템 전체가 새로 필요하다.

## 현재 상태 분석

| 항목 | 현재 상태 |
|------|----------|
| 프레임워크 | Next.js 14 (App Router, TypeScript) |
| 스타일링 | Tailwind CSS + 커스텀 다크 테마 CSS 변수 |
| UI 라이브러리 | 없음 (커스텀 컴포넌트) |
| 인증/인가 | 없음 |
| 데이터베이스 | 없음 (파일 기반 상태 관리) |
| ORM | 없음 |
| 환경변수 관리 | 없음 (.env 파일 없음) |
| 기존 페이지 | `/` (대시보드) 1개만 존재 |
| API 라우트 | pipeline 관련 4개 (start, stream, approve, outputs) |

## 영향 범위

| 영역 | 영향 | 상세 |
|------|------|------|
| **FE** | O | 로그인 페이지 UI, 폼 유효성 검사, 인증 상태 관리, 라우트 보호(미인증 시 리다이렉트) |
| **BE** | O | 인증 API 엔드포인트 (로그인/로그아웃), 세션 또는 JWT 토큰 관리, 사용자 검증 로직 |
| **DB** | O | 사용자 테이블 신규 생성 (DB 도입 필요 — 현재 DB 없음) |
| **Infra** | O | DB 컨테이너 설정, 환경변수 관리 (JWT 시크릿, DB 연결 문자열), Docker 구성 |

## 예상 실행 순서

```
PHASE 1 (순차)
  └─ 기획자(Mina): 로그인 기능 명세 작성

PHASE 2 (병렬)
  ├─ 디자이너(Lena): 로그인 화면 UI/UX 설계
  └─ BE 설계자(Sam): 인증 API 명세 + ERD 설계

PHASE 3 (병렬 + 순차 혼합)
  ├─ 인프라(Dex): DB 컨테이너 + 환경변수 설정  ← 선행
  ├─ BE(Sam): 인증 API 구현 + DB 마이그레이션   ← Infra 완료 후
  └─ FE(Jay): 로그인 페이지 UI 구현             ← 독립 착수 가능
      └─ FE(Jay): API 연동                     ← BE 완료 후

PHASE 4 (병렬)
  ├─ QA(Eva): 기능 테스트 + 엣지케이스 검증
  ├─ 보안 리뷰어(Rex): 인증 보안 감사
  └─ 코드 리뷰어(Nora): 코드 품질 검토
```

## 주요 고려사항

1. **DB 도입 결정 필요**: 현재 DB가 없으므로 경량 DB(SQLite) vs 본격 DB(PostgreSQL) 선택이 필요
2. **인증 방식 결정 필요**: JWT(Stateless) vs Session(Stateful) 중 선택
3. **회원가입 포함 여부**: 요청은 "로그인페이지"이나, 사용자 생성 방법(회원가입 페이지 or 초기 관리자 계정 시딩)을 결정해야 함
4. **기존 API 보호**: 로그인 구현 후 기존 pipeline API 라우트에도 인증 미들웨어 적용 필요
