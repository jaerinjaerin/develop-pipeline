1. 개요

목적

현재 Agent Pipeline Dashboard는 인증 없이 누구나 접근 가능하다. 로그인 페이지를 구현하여 대시보드 접근을 인증된 사용자로 제한하고, 파이프라인 실행/승인 등 핵심 기능을 보호한다.

핵심 가치

  • 접근 통제: 인증된 사용자만 대시보드와 파이프라인을 조작할 수 있다
  • 보안 기반 확보: 향후 사용자별 권한 관리, 감사 로그 등의 기반이 된다
  • 계정 안전성: 비밀번호 해싱, JWT 토큰 기반의 안전한 인증 흐름을 제공한다

작업 범위

포함:

  • 로그인 페이지 UI (이메일 + 비밀번호 폼)
  • 인증 API (로그인 / 로그아웃 / 토큰 검증)
  • JWT 기반 인증 토큰 관리
  • 사용자 테이블 (SQLite DB 도입)
  • 초기 관리자 계정 시딩 (seed 스크립트)
  • 미인증 사용자 리다이렉트 (라우트 가드)
  • 기존 pipeline API에 인증 미들웨어 적용

미포함:

  • 회원가입 페이지 (초기에는 시딩으로 관리자 계정만 생성)
  • 비밀번호 찾기/재설정
  • 소셜 로그인 (Google, GitHub 등)
  • 사용자 관리 페이지 (CRUD)
  • 역할 기반 권한 관리 (RBAC)

6. 엣지케이스 & 예외 처리

ID상황예상 동작관련 기능

E-03 JWT 토큰 만료 후 API 요청 401 응답 반환, FE에서 감지하여 로그인 페이지로 리다이렉트 F-04, F-03
E-04 이미 로그인한 사용자가 /login 접근 대시보드(/)로 리다이렉트 F-03
E-05 빈 이메일 또는 비밀번호로 제출 클라이언트 사이드에서 제출 차단, 인라인 에러 메시지 표시 F-06
E-06 잘못된 형식의 이메일 입력 클라이언트 사이드에서 이메일 형식 검증 실패 메시지 표시 F-06
E-07 JWT 시크릿 환경변수 미설정 서버 시작 시 에러 로그 출력 및 기본값 사용 방지 (앱 시작 실패) F-04
E-08 토큰이 변조된 요청 401 응답 반환, 쿠키 삭제 F-04
E-09 DB 파일 손상 또는 접근 불가 500 에러 반환, 서버 로그에 상세 에러 기록 F-07
E-10 SSE 스트림 연결 중 토큰 만료 스트림 연결 종료, 클라이언트에서 재인증 요청 F-09, F-03

7. 비기능 요구사항

항목기준비고
성능 로그인 API 응답 500ms 이내 bcrypt 해싱 비용(cost factor 10~12)을 고려한 기준
보안 비밀번호는 bcrypt로 해싱하여 저장 평문 저장 금지. salt 자동 생성
보안 JWT 토큰은 httpOnly + Secure + SameSite=Strict 쿠키에 저장 XSS로 토큰 탈취 방지
보안 JWT 만료 시간 24시간 만료 후 재로그인 필요
보안 로그인 실패 시 이메일 존재 여부 미노출 열거 공격(Enumeration Attack) 방지
보안 CSRF 방지를 위한 SameSite 쿠키 설정 Next.js 동일 출처이므로 SameSite=Strict로 충분
접근성 로그인 폼에 적절한 label, aria 속성 적용 스크린 리더 대응
접근성 키보드만으로 로그인 가능 (Tab 이동, Enter 제출) 마우스 없이 조작 가능
접근성 에러 메시지를 aria-live 영역에 표시 보조 기술에 에러 즉시 전달
UX 기존 다크 테마(#0a0a14 배경)와 일관된 디자인 기존 CSS 변수 활용
UX 로그인 중 로딩 상태 표시 (버튼 스피너) 중복 제출 방지 겸용
인프라 SQLite DB 파일은 프로젝트 루트의 data/ 디렉토리에 저장 Docker 볼륨 마운트 고려
인프라 환경변수: JWT_SECRET, ADMIN_EMAIL, ADMIN_PASSWORD .env 파일로 관리, .gitignore에 추가
