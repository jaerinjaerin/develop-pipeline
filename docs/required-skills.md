# 필요 스킬 목록

## 개요

개발 자동화 워크플로우를 구현하기 위해 각 에이전트에게 필요한 스킬을 정리한다.

## 공통 스킬

모든 에이전트가 기본으로 갖춰야 하는 스킬.

| 스킬 | 설명 |
|------|------|
| 아키텍처 문서 읽기 | `docs/architecture.md` 파악 |
| 코딩 컨벤션 문서 읽기 | `docs/coding-convention.md` 파악 |
| 이슈 내용 읽기 | 이슈 제목, 설명, 메타데이터 파악 |
| 코드베이스 읽기 | 기존 코드 탐색 및 이해 |
| 프로젝트 관리 도구 접근 | GitHub/GitLab/Jira 이슈 읽기/쓰기 |
| 에이전트 간 피드백 전달 | 구체적인 수정 사항 주고받기 |

## 에이전트별 전문 스킬

### PRD 생성 에이전트

| 스킬 | 설명 | 구현 방법 |
|------|------|----------|
| 맥락 기반 질문 생성 | 아키텍처 문서 기반 부족한 정보 질문 | LLM 프롬프팅 + 아키텍처 문서 |
| PRD 구조화 | 자연어 → 정형화된 PRD | LLM 프롬프팅 + PRD 템플릿 |
| 영향 범위 분석 | DB/BE/FE 변경 영역 식별 | 아키텍처 문서 + 코드베이스 분석 |

### 이슈 생성 에이전트

| 스킬 | 설명 | 구현 방법 |
|------|------|----------|
| 작업 분해 | PRD → area별 이슈 단위 분해 | LLM 프롬프팅 |
| 의존성 분석 | 이슈 간 선후행 관계 파악 | LLM 추론 |
| 이슈 등록 | 프로젝트 관리 도구에 생성 | GitHub CLI / MCP / API |
| 라벨링 | area, complexity 자동 부여 | LLM 판단 |

### Dev Agent

| 스킬 | 설명 | 구현 방법 |
|------|------|----------|
| 코드 작성 | area에 맞는 언어/프레임워크로 구현 | LLM 코드 생성 |
| 코드 수정 | 피드백 반영하여 수정 | LLM + diff 기반 수정 |
| 관련 파일 탐색 | 코드베이스에서 수정 대상 파일 찾기 | 코드 인덱싱 + 파일 탐색 |
| Area 컨텍스트 전환 | area에 따라 프롬프트/도구 전환 | 시스템 프롬프트 동적 로딩 |
| 커밋 생성 | Conventional Commits 규칙 | Git CLI + 커밋 메시지 생성 |

### QA Agent

| 스킬 | 설명 | 구현 방법 |
|------|------|----------|
| 테스트 케이스 도출 | 이슈 요구사항 → 테스트 케이스 | LLM 프롬프팅 |
| 유닛테스트 작성 | area별 테스트 코드 작성 | LLM 코드 생성 |
| 테스트 실행 | 테스트 명령어 실행 + 결과 수집 | CLI 실행 (npm test, pytest 등) |
| 실패 원인 분석 | 에러 로그 → 원인 파악 | LLM 분석 |
| 피드백 작성 | Dev에게 전달할 수정 방향 작성 | LLM 프롬프팅 |

### Review Agent

| 스킬 | 설명 | 구현 방법 |
|------|------|----------|
| 코드 리뷰 | 컨벤션/보안/성능/아키텍처 리뷰 | LLM + 컨벤션 문서 참조 |
| Diff 분석 | 변경된 코드만 집중 리뷰 | Git diff 파싱 + LLM |
| 테스트 품질 평가 | 커버리지와 의미 평가 | LLM 프롬프팅 |
| 통과/실패 판단 | 리뷰 기준에 따른 판정 | LLM 판단 + 체크리스트 |
| 리뷰 코멘트 작성 | 파일/라인 단위 수정 사항 | LLM 프롬프팅 |

### 오케스트레이터

| 스킬 | 설명 | 구현 방법 |
|------|------|----------|
| 의존성 그래프 관리 | depends_on 관계 추적 | DAG 자료구조 |
| 실행 순서 결정 | 병렬/순차 실행 판단 | 위상 정렬 |
| 에이전트 팀 배치 | 이슈에 Dev+QA+Review 배치 | 에이전트 스폰/관리 |
| 상태 감시 | 이슈 완료/실패 감지 | 이벤트 리스닝 |
| 에스컬레이션 | 3회 실패 시 사람 알림 | 슬랙/이메일 알림 |

### E2E Test Agent

| 스킬 | 설명 | 구현 방법 |
|------|------|----------|
| E2E 시나리오 작성 | PRD → 사용자 시나리오 도출 | LLM 프롬프팅 |
| CDP 기반 테스트 실행 | 브라우저 자동화 테스트 | Playwright / Puppeteer |
| 스크린샷 수집 | 실패 시점 브라우저 상태 캡처 | CDP 스크린샷 API |
| 실패 이슈 매핑 | E2E 실패 → 특정 이슈 매핑 | LLM 추론 + 코드 변경 이력 |
| PR 생성 | 테스트 통과 후 PR 자동 생성 | GitHub CLI / MCP |

## 도구 및 인프라 요구사항

### 필수 도구

| 도구 | 용도 |
|------|------|
| Claude Code | 에이전트 실행 엔진 |
| Git | 버전 관리, 브랜치, 커밋 |
| 프로젝트 관리 도구 | 이슈 등록/관리 (GitHub, GitLab, Jira) |
| 테스트 프레임워크 | 유닛테스트 (Jest, pytest, Vitest 등) |
| CDP 브라우저 | E2E 테스트 (Playwright, Puppeteer) |

### 연결 방법

| 연결 대상 | 방법 |
|----------|------|
| 코드베이스 | GitHub MCP / GitLab MCP / 로컬 클론 |
| 문서 | 로컬 파일 (`docs/`) |
| 이슈 관리 | GitHub CLI / GitLab MCP / Jira API |
| 알림 | Slack API / 이메일 |

### 사전 준비 문서

| 문서 | 위치 | 용도 |
|------|------|------|
| 아키텍처 문서 | `docs/architecture.md` | PRD 생성, Dev, 이슈 생성 |
| 코딩 컨벤션 문서 | `docs/coding-convention.md` | Dev, Review |
