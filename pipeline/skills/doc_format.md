# Skill: 문서 포맷 규칙 (doc_format)

## 적용 대상

Agent 01 (문서 작성), Agent 06 (에러 문서화), Agent 07 (성공 로그)가 생성하는 모든 문서

## 파일 명명 규칙

- 한글 이름 허용: `화면명세서.md`, `기능명세서.md`, `API명세초안.md`
- 에러 문서: `{YYYY-MM-DD}_{TC-ID}_{간단설명}.md`
  - 예: `2026-02-19_TC-05_login-500.md`
- QA 로그 폴더: `{YYYY-MM-DD}_{페이지명}/`
  - 예: `2026-02-19_login/`

## 문서 구조 규칙

### 화면명세서

```markdown
# {페이지명} 화면명세서

## 개요
- 페이지 경로: /{path}
- 설명: {한 줄 설명}

## 레이아웃
{전체 구조 설명}

## 컴포넌트 상세
### {컴포넌트명}
- 위치: {레이아웃 내 위치}
- 입력 필드: {필드 목록 및 스펙}
- 버튼: {버튼 목록 및 동작}
- 에러 표시: {에러 메시지 위치 및 문구}

## 반응형
| 브레이크포인트 | 변경사항 |
|---|---|
| Desktop (>1024px) | {설명} |
| Tablet (768~1024px) | {설명} |
| Mobile (<768px) | {설명} |
```

### 기능명세서

```markdown
# {기능명} 기능명세서

## 비즈니스 로직
{로직 설명}

## 분기 조건
| 조건 | 결과 |
|---|---|
| {조건} | {결과} |

## 예외 처리
| 예외 | 처리 방식 |
|---|---|
| {예외} | {처리} |

## 정책
- {정책 항목}: {값}

## 플로우차트
\`\`\`mermaid
flowchart TD
  ...
\`\`\`
```

### API명세초안

```markdown
# API 명세

## {엔드포인트명}
- Method: {GET|POST|PUT|DELETE}
- Path: /{path}
- Description: {설명}

### Request
\`\`\`json
{요청 JSON 구조}
\`\`\`

### Response
| Status | Body | 설명 |
|---|---|---|
| 200 | {응답 구조} | 성공 |
| 401 | {에러 구조} | 인증 실패 |

### 에러 코드
| Code | Message | 설명 |
|---|---|---|
| {코드} | {메시지} | {설명} |
```

## 마크다운 규칙

- 제목 레벨: `#` (문서 제목) → `##` (섹션) → `###` (서브섹션)
- 테이블: 가독성을 위해 적극 활용
- 코드 블록: 언어 명시 (json, javascript, python, mermaid 등)
- 체크리스트: Acceptance Criteria에 사용 (`- [ ]`)
